# Introduction to Log Analysis

## Objective

The objective of this lab is to understand the fundamentals of log analysis and its importance in cybersecurity. In this exercise, I explored Linux authentication logs to observe system activities, identify authentication events, and gain hands-on experience monitoring logs that are commonly analyzed by SOC analysts during security investigations.

---

## What is Log Analysis?

Log analysis is the process of collecting, reviewing, and interpreting system-generated logs to understand system activity and identify potential security events. Logs provide valuable information about user actions, authentication attempts, running services, system errors, and application behavior.

Effective log analysis helps security teams detect threats, investigate incidents, and monitor system health.

---

## Why Log Analysis Matters

* Detect unauthorized access attempts
* Monitor authentication activity
* Investigate security incidents
* Identify suspicious behavior
* Support digital forensics
* Maintain security compliance

## Command Used

```bash
sudo tail -f /var/log/auth.log
```

This command continuously displays newly generated authentication log entries in real time.

* Successful privilege escalation using `sudo`

* Invalid user login attempts
* CRON job execution
* SSH connection reset events
